# ZJU SQTP 导航网站 - 第 1 步功能设计记录

日期：2026-02-04  
阶段：功能设计（Step 1）  
目标：明确数据结构，覆盖“学科资源导航 + 资源评论 + 常用网站导航 + 在线自习室”四类需求，并区分“全员可见”与“个人私有”。

---

## 1. 过程记录

- 明确双主线导航：
  - 学科资源导航：直达帖子、视频、网盘等深度链接。
  - 常用网站导航：仅导航到网站主页（类似浏览器新标签页）。
- 明确数据可见性：
  - 学科资源投稿：未来全员可见（Demo 阶段仍用静态数据/本地模拟）。
  - 个人常用网站：仅自己可见（本地存储）。
- 评论区策略：
  - 资源预置评论写入数据文件。
  - 用户新增评论存入本地，渲染时合并展示。
  - 增加 `likes` 字段支持点赞。

---

## 2. 结果（数据结构设计）

**核心结构拆分为两条导航线：**

- 学科资源：`studyResourceCategories`
  - 结构：学科 → 资源列表 → 评论
  - 资源字段：`title`、`url`、`type`、`source`、`tags`
  - 评论字段：`id`、`user`、`content`、`time`、`likes`

- 常用网站：`siteCategories`
  - 结构：分类 → 网站主页列表
  - 字段：`title`、`url`、`description`、`iconUrl`

**可见性与未来扩展：**

- `userCustomSites`：个人常用网站（本地存储容器）。
- `futureShared`：标记“学习资料投稿未来将同步后端”。
- `studyRoom`：在线自习室配置（番茄钟参数 + 在线人数占位）。

---

## 3. 交付物

- 数据结构文件： [data/data.json](data/data.json)
- 本地存储规划（口头约定）：
  - `resourceCommentsById`（资源评论映射）
  - `userCustomSites`（个人常用网站）
  - `userSubmittedResources`（学习资料投稿，待同步）

# 第 2 步：UI 设计记录

日期：2026-02-04  
阶段：UI 设计（Step 2）  
目标：确定页面布局、视觉风格与关键组件交互规范。

---

## 1. 过程记录

- 采用三栏布局：左侧导航 / 中间内容 / 右侧自习室。
- 明确双导航线：学科资源（深度链接）与常用网站（主页导航）。
- 明确交互入口：资源卡片含“打开/评论”，常用网站卡片含“打开主页”。
- 自习室独立区域：番茄钟 + 在线人数。

---

## 2. 结果（布局与组件）

**布局结构（模块级）：**

- `header`：站点名 + 副标题
- `main`：三栏布局
  - `aside.nav`：分类导航 + “+ 添加网站”
  - `section.content`：资源/网站卡片区
  - `aside.study-room`：番茄钟与在线人数
- `footer`：署名/版权

**组件要点：**

- 左侧导航：选中态高亮 + 左侧色条
- 资源卡片：标题、来源标签、简介、打开/评论
- 网站卡片：Logo/首字母 + 站点名 + 打开主页
- 自习室：大号计时 + 开始/暂停 + 切换模式

---

## 3. 视觉规范（设计口径）

- 主题色：#2B5ECF（主色）、#F2F6FF（浅背景）、#FFB25B（点缀）
- 字体建议：标题用衬线（思源宋体），正文用无衬线（思源黑体）
- 卡片：圆角 12px，轻阴影，悬浮抬升
- 标签：胶囊样式，浅底深字
- 按钮：主按钮实色，次按钮描边
- 间距：卡片内边距 16–20px，栏间距 24–32px
